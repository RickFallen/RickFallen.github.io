思考：Spring 启动后如何修改 Bean 对象的属性值

前言

在复习 Spring 知识时突然想到一个问题，在 Spring 初始化过程中，我们可以使用 BeanPostProcessor 在对象初始化前后、实例化前后、属性设值之后修改 bean 的属性值。那当 Spring 容器已经启动之后呢？如何修改 Bean 对象的属性值

解决方案

  1. 在 Bean 方法中写 reload 方法

    当接收到触发修改 bean 的事件时，通过 SpringContext 拿到对应的 Bean，然后调用 Bean 对象事先写好的 reload 方法修改对象。由于 Bean 实例的引用存在 Spring 容器中，所以只要这一处修改处处生效

  2. 在 Bean Registry 中删除并重新注册 Bean，但缺点是已经持有修改 Bean 之前的对象不会更新属性值

  3. @RefreshScope
